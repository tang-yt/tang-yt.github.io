# vercel

vercel加速hexo+github部署的博客

重庆

目前大多数小伙伴都选用 hexo 来搭建我们的个人博客，并且使用 gitee pages 或 github pages 来托管我们的页面，但是这样的访问速度是很慢的。也看了很多博主分享，最终自己还是选用了Vercel，还可以通过https访问，最后结果,个人感觉确实速度快了不少。

使用教程

1. 基本教程，使用 Vercel 默认域名

① 进入Vercel官网，我们选择 github 登录。

② 进来后，我们选择创建项目。

③ 选择自己hexo博客的仓库，点击 Import。

④ 接下来就是做一些自定义配置，我们选择一个 工程名 然后直接点击部署即可。

⑤ 构建成功后，我们可以在控制台看到 Vercel 提供了一个默认域名来访问，而且还是自带 https 的，作为一名白嫖党表示很高兴，访问速度还是挺快的：（但是还是要 课雪上网，后面也提供了相应的代理）。

2. 绑定自定义域名（以阿里云域名为例）

① 点击 Domains，进入控制台。

② 输入自己的域名，点击 Add。

这里我添加的是 tangyt.cn 这个域名，Vercel 默认提供了一个选项（同时增加域名 www.tangyt.cn 重定向到 tangyt.cn），我们直接选择即可。

③ 添加后，应该都是显示 Invalid Configuration，因为我们还没有配置解析，直接去 阿里云域名控制台设置解析即可。

3. 配置解析

因为我刚才勾选了 （同时增加域名 www.tangyt.cn 重定向到 tangyt.cn）选项，所以需要同时设置两个解析，设置的解析按照 Vercel 提供的选项即可。

  • 首先我们配置 tangyt.cn 的解析

  • 再配置 www.tangyt.cn 的解析

非常直观，直接各值一一对应添加即可。

添加解析完之后，我们再回到 Vercel 的页面，发现已经配置成功了。

现在，我们就可以使用我们自己添加的域名进行访问了。

4. Vercel域名在国内被墙

Vercel 因为被大量使用，自然而然被墙掉了，不过好在 Vercel 官方提供了单独的 IP 和 CNAME 地址给大家，对于国内的用户来说，配置一下单独的解析，依然可以享受 Vercel 提供的服务。

将上述步骤中用到的 ip 和 cname 地址替换成以下内容即可：

  • A记录地址：76.223.126.88 或 76.76.21.98 等

  • CNAME 记录地址：cname-china.vercel-dns.com
