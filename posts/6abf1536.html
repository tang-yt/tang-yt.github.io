SpringBoot配置文件

西安

一、配置文件简介

整个项目中所有重要的数据都是在配置文件中配置的，比如：
① 数据库的连接信息
② 项目的启动端口
③ 第三方系统的调用密钥等信息
④ 用于发现和定位问题的普通日志和异常日志
…

  • SpringBoot项目是一个标准的Maven项目，它的配置文件需要放在src/main/resources/下，其文件名必须为application，其存在两种文件形式，分别是properties、yaml（或yml）文件。

二、properties 配置文件说明

  • properties 是早期的配置文件的格式，也是 Spring Boot 项目创建后默认的配置文件格式。
  • properties 以键值对的形式配置的，key 和 value 之间用 = 号连接。key 就是要配置的属性，value 就是要设置的值。

三、yaml 配置文件说明

  • YAML（YAML Ain’t a Markup Language）是一种可读性高、易于理解、用于表达数据序列化的格式
  • 优点：
    ① 容易阅读
    ② 容易与脚本语言交互
    ③ 以数据为核心，重视数据轻格式
  • yaml 文件拓展名
    ① .yml（主流），是 yaml 的缩写
    ② .yaml

1. yaml 语法规则

  • 大小写敏感
  • 属性层次关系使用多行描述，每行结尾使用 :结束
  • 使用缩进表示层次关系，同层级左侧对齐，只允许使用空格（不允许会用 Tab 键）
  • 属性值前面添加空格（属性名与属性值之间使用 冒号 + 空格 作为分割）
  • # 表示注释

四、properties 和 yaml 的总结

  • properties与yaml语法结构
properties yaml或yml
语法结构 key=value key: value（:和value之间需要空格）
  • properties 为早期并且默认的配置⽂件格式，但其配置存在⼀定的冗余数据，使⽤ yaml 可以很好的 解决数据冗余的问题
  • yaml 通⽤性更好，⽀持更多语⾔，如 Java、Go、Python 等，如果是云服务器开发，可以使⽤⼀份配置⽂件作为 Java 和 Go 的共同配置⽂件
  • yaml 虽然可以和 properties 共存，但⼀个项⽬中建议使⽤统⼀的配置类型⽂件
  • yaml 支持更多的数据类型
  • 运行优先级： properties > yml > yaml
